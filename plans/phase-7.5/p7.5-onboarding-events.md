# Phase 7.5: Onboarding Events Step — ✅ Complete (PR #128)

## Goal

Add a new onboarding step that lets users add their key events (races, camps, travel) for the year. This gives the AI coach critical context for periodization and plan generation from day one. Users with Intervals.icu connected should be able to import events directly; others can add events manually.

**Discovered during:** OB-15/OB-18 manual testing (user feedback: "part of the onboarding should be adding the events for the year")
**Priority:** Medium - improves plan quality significantly but not a blocker

---

## Current State

- Onboarding flow: Welcome -> Intervals.icu -> Fitness Numbers -> Goals -> Training Plan -> Done
- Events/calendar data is only available if the user connects Intervals.icu
- The calendar tab (`apps/mobile/app/(tabs)/calendar.tsx`) already displays events from Intervals.icu
- `mcp-gateway` already has `getEvents` tool to fetch events from Intervals.icu API
- No manual event creation UI exists anywhere in the app
- Training plan generation has no knowledge of upcoming races/events

## Proposed Approach

### New Onboarding Screen: Events (between Goals and Training Plan)

1. **If Intervals.icu connected:** Auto-import events from Intervals.icu for the next 12 months. Show them in a list with checkmarks. User can deselect events they don't care about.
2. **If no Intervals.icu:** Show manual event entry form (event name, type, date, priority). Allow adding multiple events.
3. **Skip option:** "Skip for now" button to proceed without events.
4. **Persist:** Save selected/entered events to a new `athlete_events` table (or reuse the calendar events pattern).

### Why This Matters

- Training plan screen comes right after events - the plan should be informed by race dates
- The AI coach needs to know about races to suggest proper taper/peak timing
- Without events, the structured plan is generic rather than periodized around key dates

## Schema Considerations

- May need an `events` table or extend `goals` with event-specific fields
- Events from Intervals.icu are transient (API-fetched) vs user-entered events that should persist
- Consider whether events should link to race goals

## Files Likely Involved

- `apps/mobile/app/onboarding/events.tsx` (new screen)
- `apps/mobile/app/onboarding/_layout.tsx` (add route)
- `apps/mobile/contexts/OnboardingContext.tsx` (add events to data model)
- `apps/mobile/services/onboarding.ts` (persist events)

## Out of Scope

- Calendar sync (already exists via Intervals.icu integration)
- Event editing after onboarding (profile/settings feature)
- Recurring events
