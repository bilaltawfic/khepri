# P7.5-DASH: Fix Intervals.icu Connection Status on Profile Page

## Goal

Fix the profile page so that Intervals.icu connection status accurately reflects whether credentials are configured, the connect button navigates to the settings screen, and users can disconnect from the profile flow.

**Discovered during:** DASH-04 manual testing (PR #130)

## Root Cause

Two bugs in `apps/mobile/app/(tabs)/profile.tsx`:

1. **Hardcoded mock data:** `athlete.intervalsIcuConnected` is always `false` (line 25). The screen never calls `useIntervalsConnection` hook which checks real credentials via the `credentials` Edge Function.
2. **Missing navigation:** The Intervals.icu `MenuItem` (lines 209-214) has no `href` prop, so tapping it does nothing. It should navigate to `/profile/intervals`.

The intervals settings screen (`apps/mobile/app/profile/intervals.tsx`) already works correctly — it uses `useIntervalsConnection` hook and supports connect/disconnect flows.

## Files to Modify

| File | Change |
|------|--------|
| `apps/mobile/app/(tabs)/profile.tsx` | Import and use `useIntervalsConnection` hook; add `href` to Intervals.icu MenuItem |
| `apps/mobile/app/(tabs)/__tests__/profile.test.tsx` | Add/update tests for connection status display and navigation |

## Implementation Steps

### 1. Import `useIntervalsConnection` hook

```typescript
import { useIntervalsConnection } from '@/hooks/useIntervalsConnection';
```

### 2. Use the hook in `ProfileScreen`

Inside `ProfileScreen()`, add:

```typescript
const { status, isLoading } = useIntervalsConnection();
```

### 3. Compute connection subtitle

Replace the hardcoded `athlete.intervalsIcuConnected` check:

```typescript
function getIntervalsSubtitle(connected: boolean, isLoading: boolean): string {
  if (isLoading) return 'Checking...';
  return connected ? 'Connected' : 'Not connected';
}
```

### 4. Add `href` to MenuItem

```typescript
<MenuItem
  icon="link-outline"
  title="Intervals.icu"
  subtitle={getIntervalsSubtitle(status.connected, isLoading)}
  colorScheme={colorScheme}
  href="/profile/intervals"
/>
```

### 5. Remove unused mock field

Remove `intervalsIcuConnected: false` from `mockAthlete` since it's no longer used.

## Testing Requirements

3. **Shows "Checking..." while loading** — Mock hook to return `{ isLoading: true }`, verify loading state
4. **Intervals.icu item navigates to settings** — Verify the MenuItem renders with correct href
