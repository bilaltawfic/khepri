## Testing Approach

1. Use `/test-engineer DASH` skill to guide user through each test case
2. For each failure:
   - Identify root cause in the component
   - Implement fix with unit test
   - Run `pnpm test` and `pnpm lint`
3. Commit all fixes together
4. Create conversation log in `claude-convos/YYYY-MM-DD/`
5. Push and create PR

## Prerequisites

- Hosted Supabase deployed with latest migrations
- Expo Go running on physical device
- User has test account with:
  - Active season (for DASH-02, 03-09)
  - Locked block with workouts (for DASH-03-09)
  - Intervals.icu connected (for DASH-06, 14)
