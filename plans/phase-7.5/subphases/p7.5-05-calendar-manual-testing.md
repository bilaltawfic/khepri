# P7.5-05: Calendar Manual Testing

## Goal

Systematically test all calendar features (CAL-01 through CAL-07) on a physical device via Expo Go connected to hosted Supabase. Fix any failures immediately and create a single fix PR.

## Test Cases (7 untested)

### Event Display
| ID | Scenario | Key Verification |
|----|----------|-----------------|
| CAL-01 | Calendar loads with events | Events grouped by date, each shows type icon, name, category, planned duration, planned TSS |
| CAL-05 | Event types display correctly | Correct icons: workout (barbell), race (trophy), rest day (bed), note (document), travel (airplane). Priority badges shown |

### Navigation
| ID | Scenario | Key Verification |
|----|----------|-----------------|
| CAL-02 | Navigation forward | Forward chevron advances 2 weeks, header updates |
| CAL-03 | Navigation backward | Back chevron goes back 2 weeks |

### Edge Cases
| ID | Scenario | Key Verification |
|----|----------|-----------------|
| CAL-04 | Empty state | No events message for empty 2-week period |
| CAL-06 | Pull to refresh | Loading indicator, events reload from Intervals.icu |
| CAL-07 | No Intervals.icu | Empty state or prompt to connect |

## Files Under Test

- `apps/mobile/app/(tabs)/calendar.tsx` — Main calendar screen
- Related hooks/services that fetch events from Intervals.icu via MCP gateway

## Testing Approach

1. Use `/test-engineer CAL` skill to guide user through each test case
2. For each failure:
   - Identify root cause in the component or data fetching layer
   - Implement fix with unit test
   - Run `pnpm test` and `pnpm lint`
3. Commit all fixes together
4. Create conversation log in `claude-convos/YYYY-MM-DD/`
5. Push and create PR

## Prerequisites

- Hosted Supabase deployed with latest migrations
- Expo Go running on physical device
- User has test account with:
  - Intervals.icu connected with upcoming events (for CAL-01, 02, 03, 05, 06)
  - Various event types in calendar (for CAL-05)
  - A 2-week period with no events accessible (for CAL-04)
  - Test without Intervals.icu connection (for CAL-07) — may need a second test account or disconnect temporarily
