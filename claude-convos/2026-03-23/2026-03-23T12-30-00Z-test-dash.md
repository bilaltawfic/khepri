## Fixes Applied

### 1. AI recommendation error fallback (`fdefaa2`)
- **Test:** DASH-03
- **Failure:** "Edge Function returned a non-2xx status code" shown to user after check-in
- **Root cause:** AI coach edge function failure was surfaced as raw error to user
- **Fix:** Fall back to local `generateMockRecommendation()` when AI coach fails; show user-friendly error message for check-in save failures

### 2. Sleep score prefill (`e5ecc80`, `fda82d4`)
- **Test:** DASH-03
- **Failure:** Intervals.icu sleep quality (1-4 scale) was incorrectly mapped, showing "1" for great sleep
- **Root cause:** Code assumed 1-5 scale; Intervals.icu uses 1-4 (1=Great, 4=Poor)
- **Fix:** Use `sleepScore` (0-100 device score) divided by 10 as primary, fall back to `sleepQuality` (1-4 inverted) when sleepScore unavailable. Required adding sleepScore to WellnessDataPoint interface and MCP gateway transform.

### 3. Remove sleep hours input (`d19a372`)
- **Test:** DASH-03
- **Failure:** Hours display was cut off and oversized (fontSize 32)
- **Fix:** Removed HoursInput from form UI entirely. Sleep hours still populated behind the scenes from Intervals.icu prefill and passed to AI coach.

### 4. Round training metrics to 2dp (`967d994`)
- **Test:** DASH-05
- **Failure:** CTL/ATL/TSB showed too many decimal places
- **Fix:** `formatMetricValue()` now uses `toFixed(2)` for non-integer values

### 5. Add form status badge (`4ad9a62`)
- **Test:** DASH-05 (enhancement)
- **Fix:** Added colored badge (Fresh/Optimal/Fatigued) with description based on TSB value. Shows "TSB = CTL − ATL" explanation.

### 6. TSB rounding consistency (`889a60d`, `0988952`)
- **Test:** DASH-05
- **Failure:** Displayed TSB didn't match CTL - ATL visually (backend rounded to 1dp, frontend displayed 2dp)
- **Fix:** Backend now rounds TSB to 2dp (`* 100 / 100`)

### 7. Remove FTP from Training Load card (`f7c7146`)
- **Test:** DASH-05
- **Failure:** FTP is a fitness threshold, not a training load metric
- **Fix:** Removed FTP display; changed empty state check from `ftp == null` to `ctl == null`

### 8. Show Intervals.icu calendar events (`86b48a3`, `c2a8786`)
- **Test:** DASH-08
- **Failure:** Upcoming Events card only showed local goals, not Intervals.icu events
- **Fix:** Fetch calendar events via `getCalendarEvents()`, merge with goals, sort by date. Extended range to 180 days. Added weeks-away labels (today, tomorrow, Xd, Xw).

### 9. Log calendar fetch failures (`50f0ae2`)
- **Fix:** Added `console.warn` for calendar event fetch errors instead of silently swallowing

### 10. Filter out mock data when Intervals.icu disconnected (`933f3b6`)
- **Test:** DASH-04, DASH-06
- **Failure:** Training Load and Recent Activities cards showed mock numbers instead of '--' when Intervals.icu was disconnected
- **Root cause:** MCP gateway returns mock data with `source: 'mock'` when no credentials configured; mobile services didn't check the source field
- **Fix:** Added `source === 'mock'` check to `getWellnessSummary()`, `getRecentActivities()`, `getCalendarEvents()`, `getWellnessData()`, and `getTodayWellness()` — all return empty/null for mock data

## Learnings

- Intervals.icu uses 1-4 scales (not 1-5) for subjective metrics, with 1=best for sleep/mood and 1=lowest for stress/fatigue/soreness
- `sleepScore` (0-100 device-reported) is more reliable than `sleepQuality` (1-4 subjective) for prefill
- Backend rounding must match frontend display precision to avoid confusing discrepancies
- FTP is a fitness threshold, not a training load metric — belongs on profile page
- The MCP gateway's MAX_DAYS limits need to account for race events months in advance
- Profile page shows "Not connected" for Intervals.icu even when it IS connected — fixed in PR #131
- MCP gateway returns mock data with `source: 'mock'` when no credentials; client must filter this to show proper empty states
