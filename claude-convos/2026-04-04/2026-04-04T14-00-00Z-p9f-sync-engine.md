# P9-F: Intervals.icu Sync Engine Implementation

**Date:** 2026-04-04
**Task:** P9-F-01 through P9-F-05

## Goals

Build the bidirectional Intervals.icu sync system per ADR-001:

1. Shared sync engine module (activity matching, compliance computation, event diff detection)
2. Push module (convert Khepri workouts to Intervals.icu events, bulk upsert)
3. Webhook Edge Function (receive real-time updates from Intervals.icu)
4. Cron reconciliation Edge Function (30-min polling safety net)
5. DSL validation with fallback for invalid workout descriptions

## Key Decisions

- **Shared module pattern**: Created `_shared/intervals-sync-engine.ts` and `_shared/intervals-push.ts` for pure logic that both webhook and cron functions share, avoiding code duplication
- **Dependency injection for push**: The `pushBlockToIntervals` function accepts an `UpsertEventFn` parameter rather than importing the API client directly, enabling clean unit testing without Deno/fetch mocking
- **Lightweight DSL validation**: Implemented heuristic validation (checks for section headers + step lines) rather than a full parser, with fallback to simplified descriptions
- **Wellness deferred to cron**: Webhook logs wellness updates but defers actual processing to cron reconciliation for simplicity
- **No deletion on event.delete**: When Intervals.icu deletes an event, the local workout is marked as `conflict` rather than deleted, preserving the training plan
- **Database types updated**: Added `sync_log` table and athlete sync columns to `packages/supabase-client/src/types.ts`
