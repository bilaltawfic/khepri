# P9-A-08 to P9-A-11: Supabase Client Queries

## Date
2026-04-04

## Goals
- Add TypeScript query functions for season-based planning tables (seasons, race_blocks, workouts, plan_adaptations) to @khepri/supabase-client
- Follow existing patterns from training-plans.ts

## Key Decisions
- Used same `QueryResult<T>` + `createError()` pattern as existing queries
- Used `limit(1)` + `data?.[0] ?? null` for optional single-row queries (getActiveSeason, getActiveBlock, getWorkoutByExternalId)
- Used `as never` cast on insert/update to work around Supabase client type strictness
- Named season lifecycle functions `archiveSeasonRecord`/`completeSeasonRecord` to avoid conflicts with existing exports
- Used `readonly` on bulk insert parameter type
- Added `RolledBackBy` type for plan adaptation rollback actor

## Learnings
- The `.then()` chaining pattern on Supabase queries returns `PromiseLike` not `Promise`, causing TS2739 errors. Use `async/await` instead.
- Biome's `noThenProperty` rule prevents adding `then` to mock objects — use local mocks with separate setup for bulk operations.
