## Fixes Applied

### Fix 1: Remove per-field Synced badges (OB-07)
- **Failure:** Per-field "Synced" badges were redundant with the top success banner
- **Root cause:** Original implementation added badges to each field; UX review found top banner sufficient
- **Fix:** Removed per-field synced badge rendering from fitness screen
- **Commit:** `9680fbb`

### Fix 2: Persist training plan to database (OB-18)
- **Failure:** Training plan selection during onboarding was not saved to the `training_plans` table
- **Root cause:** `saveOnboardingData` did not call `createTrainingPlan` — the `planDurationWeeks` field was collected but never persisted
- **Fix:** Added step 4 to `saveOnboardingData` that creates a training plan row with `athlete_id`, `name`, `total_weeks`, `start_date`, and `end_date`. Added `formatDate` helper. Added 3 unit tests.
- **Commit:** `bdeeb8e`

### Fix 3: Pre-populate goals on re-entry (OB-18)
- **Failure:** When re-entering the goals screen (e.g., re-running onboarding), previously saved goals were not shown
- **Root cause:** Goals screen only used in-memory OnboardingContext state; no DB fetch on mount
- **Fix:** Added `useEffect` to goals screen that fetches existing active goals from the database via `getActiveGoals` and seeds the OnboardingContext. Added runtime validation (`isValidGoalType`, `isValidPriority`) before type assertions. Updated tests with proper mocks.
- **Commit:** `9d538f1`

### Fix 4: Prevent duplicate goals on onboarding re-run
- **Failure:** Running onboarding multiple times created duplicate goals in the database
- **Root cause:** `saveOnboardingData` always inserted new goals without clearing existing ones
- **Fix:** Added `deleteActiveGoals` query function. `saveOnboardingData` now deletes existing active goals before inserting the new set from the onboarding context. Added 5 unit tests (3 in mobile, 2 in supabase-client).
- **Commit:** (included in this PR)

### Deployment Fix: Credentials edge function (OB-11b)
- **Issue:** Invalid Intervals.icu credentials were accepted on the connect screen
- **Root cause:** The `credentials` edge function had not been redeployed since PR #123 added server-side validation
- **Fix:** Redeployed with `npx supabase functions deploy credentials --no-verify-jwt`
- **No code change required**

## Learnings

- Edge function deployments can fall out of sync with code changes — always verify deployment status when testing server-side features
- Training plan schema has a validation trigger (`validate_training_plan_weeks`) that checks `total_weeks` matches the date range within +/-1 week tolerance
- Runtime validation of external data (goal types, priorities) prevents invalid type assertions when reading from the database
- The `useRef` guard pattern (`hasFetchedGoals.current`) prevents duplicate DB fetches in React strict mode
