- ScreenContainer had double padding on chat tab

- Tab screens had inconsistent top padding
- Coach didn't have context about today's check-in
**Fix Applied:**

- Added check-in context as coach system message with wellness scores
- Bottom-aligned chat messages
- Fixed top padding across all tab screens (ScreenContainer edges=[])
- Added chat tests for check-in context display
**Commit:** `fix(mobile): address CI-01 through CI-06 manual test failures`

**Failure:** Local fallback didn't show fallback notice or pass athlete constraints

- Aligned threshold values across AI prompts
**Commit:** `fix(mobile): pass athlete constraints to local fallback and show fallback notice`

## Learnings

1. **JSONB data needs runtime validation**: AI recommendation data stored as JSONB must be validated before type assertion since the shape isn't guaranteed by the database schema
2. **Manual mocks need maintenance**: When adding new query functions to `@khepri/supabase-client`, the manual mock at `__mocks__/@khepri/supabase-client.ts` must also be updated
3. **`jest.clearAllMocks()` resets mock implementations**: Tests using manual mocks need a `resetMocks()` helper in `beforeEach` to re-establish mock return values after `clearAllMocks()`
4. **Real data connections improve UX significantly**: Moving from mock to real data in the history screen, combined with the detail view, made the check-in flow feel complete
5. **Consistent threshold values across prompts matter**: AI coach and orchestrator prompts should use the same threshold values for wellness score interpretation
6. **Global jest.setup.ts mocks override test file mocks**: When `jest.setup.ts` mocks `expo-router` globally, test files must use `jest.mocked()` to override specific return values rather than re-mocking the module
7. **Extract shared utils to reduce SonarCloud duplication**: Identical helper functions across files trigger the duplicated lines quality gate — extract to shared utils with tests
