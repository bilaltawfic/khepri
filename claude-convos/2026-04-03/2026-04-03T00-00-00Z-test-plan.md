## Fixes Applied

### 1. SafeAreaView edges for tab screens
- **Issue:** Tab screens had double safe-area padding from the tab navigator
- **Fix:** Added `edges={['left', 'right']}` to ScreenContainer in all tab screens
- **Files:** calendar.tsx, checkin.tsx, index.tsx, profile.tsx

### 2. Onboarding plan missing periodization data
- **Issue:** Plans created during onboarding had NULL periodization, causing empty phases/volume on plan screen
- **Root cause:** `saveTrainingPlan()` in onboarding service didn't call `generatePeriodizationPlan()`
- **Fix:** Added periodization generation to onboarding + migration 008 to backfill and add NOT NULL constraint
- **Files:** onboarding.ts, onboarding.test.ts, migration 008

### 3. Plan screen improvements
- **Issue:** Empty state directed users to chat instead of providing a create form; no date context on phases/volume
- **Fix:** Replaced EmptyState with inline CreatePlanForm with duration picker chips; added date ranges to phase timeline and volume chart headers
- **Files:** plan.tsx, plan.test.tsx

### 4. Remove pause functionality (PLAN-05)
- **Issue:** Pause doesn't make training sense — periodization is time-bound, if you miss a week you resume at the current point
- **Fix:** Removed pause from all layers: DB constraint (migration 009), PlanStatus type, pauseTrainingPlan query, useTrainingPlan hook, plan screen UI, and all tests
- **Files:** types.ts, training-plans.ts, queries/index.ts, supabase-client/index.ts, useTrainingPlan.ts, plan.tsx, + test files, migration 009

### 5. Cancel dialog copy (PLAN-06)
- **Issue:** Dialog said "This cannot be undone" which is technically false (status could be reactivated)
- **Fix:** Updated to "Your plan will be cancelled and you'll need to create a new one to continue training."
- **Files:** plan.tsx

## Learnings
- Pause doesn't make sense for periodized training plans — the timeline is fixed and athletes resume where they are
- Onboarding and Plan tab had divergent plan creation paths; both now generate periodization data
- Date ranges on phases/volume charts greatly improve context for the user
- Volume multipliers are a UX gap — future work should explore absolute hours or better volume representation
