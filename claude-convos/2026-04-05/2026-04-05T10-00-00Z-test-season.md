## Fixes Applied (This Session — Conversations 2 & 3)

### 1. Persist season setup data to AsyncStorage (0e18ad5)
- **Test:** SEASON-07 — data lost on hot reload
- **Root cause:** SeasonSetupContext only used React useState; hot reloads remounted provider
- **Fix:** Added AsyncStorage persistence with hydration on mount, debounced saves, clear on reset

### 2. Infer race distance from name during import (3d1947f)
- **Test:** SEASON-08 — imported races got `Custom` distance
- **Root cause:** Hardcoded `distance: 'Custom'` for all imports
- **Fix:** Added `inferDistance()` with regex patterns matching race names to known distances

### 3. Re-import updates existing races instead of skipping (656f4d8)
- **Test:** SEASON-03 — re-import skipped existing races
- **Root cause:** Dedup logic filtered out matching keys entirely
- **Fix:** Map-based update strategy: update existing by `name::date` key, append new ones

### 4. Rename 70.3 to Ironman 70.3 and add T100 distance (d5cc86f)
- **Test:** SEASON-08 — consistency and completeness
- **Fix:** Renamed in RACE_DISTANCES and MIN_HOURS_BY_RACE, added T100 with 8h minimum

### 5. Don't show hours warning on empty or zero max field (503858c)
- **Test:** SEASON-08 — "0h/week" warning on empty max
- **Root cause:** `Number('')` returns 0
- **Fix:** Check for empty trimmed string before numeric conversion

### 6. Improve hours warning with min/max validation (5e19113)
- **Test:** SEASON-08 — max < min not validated in real-time
- **Fix:** Added min > max comparison to `getHoursWarning` with clear message

### 7. Update loading message to set 1-2 minute expectation (2b46d1c)
- **Test:** SEASON-09 — user feedback on generation time
- **Fix:** Changed subtitle to "This usually takes 1-2 minutes"

### 8. Remove regenerate button from season overview (5f3e625)
- **Test:** SEASON-10 — regeneration without guided input produces random results
- **Fix:** Removed button, tracked "regenerate with adjustment notes" as future improvement

## Learnings
- AsyncStorage is essential for wizard flows in React Native — hot reloads lose all useState
- Name-based distance inference works well for standard race names but Intervals.icu doesn't categorize triathlons specifically
- Real-time form validation (during typing) is much better UX than validation only on submit
- Regeneration without user input about what to adjust is not useful — needs guided input (future improvement)
