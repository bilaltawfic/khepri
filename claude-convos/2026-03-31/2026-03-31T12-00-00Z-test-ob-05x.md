## Fixes Applied

### OB-05a: Invalid credentials error

**Failure:** Generic "Failed to save credentials" error shown instead of specific error message from the server. Also, iOS prompted to save credentials as a password. Profile screen did not match onboarding screen error styling.

**Root Cause (multi-part):**
1. Edge function needed redeployment to return specific error messages
2. Profile screen (`intervals.tsx`) used hardcoded `Alert.alert('Error', 'Failed to save connection settings')` — discarding the actual error message from `saveCredentials`
3. `secureTextEntry` on API key input triggered iOS password autofill
4. Profile screen inputs didn't show red borders on connection error like onboarding screen

**Fix Applied:**
- **`credentials.ts`**: Added status-code-specific fallback messages for 401, 429, 502 when the server response has no JSON body
- **`credentials.test.ts`**: Added 3 new tests for status-code fallbacks; updated existing fallback test
- **`intervals.tsx`**: Replaced Alert dialog with inline error banner (icon + text), added red border on inputs via error prop, removed `secureTextEntry`, added `textContentType="none"`, added error state clearing on field edit
- **`connect.tsx`**: Removed `secureTextEntry`, added `textContentType="none"` to both inputs
- **`FormInput.tsx`**: Changed error text rendering to skip whitespace-only errors (allows passing `' '` to trigger red border without showing text)
- Redeployed credentials edge function

## Learnings

- Always check which screen the user is testing from — onboarding and profile screens have independent implementations
- iOS `secureTextEntry` triggers password autofill; use `textContentType="none"` to suppress
- Passing whitespace-only error strings to FormInput is an effective pattern for triggering visual error state (red borders) without showing error text
- Edge functions need redeployment after changes — stale deployments can mask server-side fixes
- Inline error banners with icons provide better UX than Alert dialogs for form validation errors
